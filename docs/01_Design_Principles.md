# C3 BASIC COMPUTER

## Design Principles

### 1. Computer First

C3 BASIC COMPUTER is a computer.

The microcontroller is an implementation detail.

---

### 2. Instant On

Power on.

READY.

Start creating.

---

### 3. Programming First

Programming is the primary interface.

Every feature should support creating, exploring, and understanding software.

---

### 4. Playable

The system should invite experimentation.

The first program should take minutes, not hours.

---

### 5. Hackable

Every practical layer of the system should be open to exploration.

Restrictions exist only to preserve recoverability.

---

### 6. Recoverable

Exploration should never permanently damage the computer.

The system software is protected.

The user workspace is renewable.

---

### 7. Transparent

The system should reveal how it works.

Whenever practical, software should expose—not hide—the underlying concepts.

---

### 8. Companion

The computer communicates clearly.

It encourages exploration.

It never gets in the maker's way.

---

### 9. Honor Computing Traditions

Preserve timeless ideas when they continue to help understanding.

READY.

HELLO, WORLD!

ERROR.

HELP.

---

### 10. Build for Makers

The system is designed for people who build, modify, experiment, and share.

---

### 11. Reference, Not Restriction

This project provides a reference implementation.

Others are encouraged to modify, extend, and experiment with it.

---

### 12. Keep It Small

Prefer simple systems over complicated ones.

A feature is complete when it is understandable.

Keep Going.
